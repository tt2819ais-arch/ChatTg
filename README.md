# 🤖 Telegram Brainstorm Bots — команда ИИ, которая штурмует твою идею

Прототип: групповой чат в Telegram, где **3–4 ИИ-бота** с разными характерами
(Стратег, Креатор, Критик, Реалист) обсуждают идею, которую ты кидаешь в чат.
Они говорят **простым человеческим языком**, задают тебе уточняющие вопросы и
**слушаются тебя в первую очередь**.

### 🚀 Как это работает (главный сценарий)

1. **Кидаешь идею** в чат — команда обсуждает её просто и по делу, задаёт вопросы.
2. **Говоришь «погнали» / «делаем»** — старший (**Стратег**, лидер) раздаёт всем
   конкретные задачи, команда обсуждает дальше.
3. **Спорные вопросы решаются голосованием** — каждый бот голосует, побеждает
   вариант с большинством голосов («где больше голосов — то и делаем»). При ничьей
   решает лидер. **Твоё слово всегда важнее любого голосования.**
4. **На выходе — НЕ код**, а два документа: **🎯 подробный ПРОМПТ** (для нейросети,
   которая потом сделает проект) и **📋 ТЗ** (техническое задание) по твоей идее.
   Их собирает и публикует Стратег.

> Слова-команды («погнали», «делаем», «го», «за дело»…) настраиваются в
> `config.yaml` → `discussion.go_words`. Лидер задаётся в `discussion.leader_name`.

---

## ⚠️ Главное про Telegram (почему всё устроено так)

Бот в Telegram **НЕ получает сообщения, которые отправил другой бот**. То есть
боты физически не могут «слышать» друг друга через Telegram и переписываться
сами.

Поэтому здесь **один центральный Python-сервис** держит все 4 токена сразу:
- ведёт **один общий транскрипт** диалога;
- по очереди спрашивает каждый ИИ, выдавая ему всю историю + его личную роль;
- постит каждую реплику в группу **под токеном нужного бота**.

Так в чате выглядит, будто 4 бота переписываются между собой, хотя на самом
деле их «мозгом» и «дирижёром» управляет один сервис. Слушать сообщения группы
достаточно **одному** боту (`listener: true`) — остальные только отправляют.

## 1️⃣ Создаём 4 ботов у @BotFather

В Telegram открой **@BotFather** и для **каждого** из 4 ботов:

1. `/newbot` → задай имя и username (напр. `My Strateg Bot`, `my_strateg_bot`).
2. BotFather пришлёт **токен** вида `123456:ABC-...` — сохрани его.
3. **Отключи приватный режим**, чтобы бот видел сообщения группы:
   `/setprivacy` → выбери этого бота → **Disable**.
4. Повтори для всех 4 ботов.

Затем:
- Создай **группу** в Telegram, добавь в неё **все 4 бота** и **себя**.
- Сделай ботов **админами** группы (по желанию, но так надёжнее — особенно если
  приватный режим где-то остался включён).

> 💡 Свой числовой Telegram **id** узнаешь у @userinfobot — он уже прописан в
> конфиге (`boss.telegram_id`), но проверь, что он правильный.

## 2️⃣ Берём ключ OpenRouter

1. Зайди на https://openrouter.ai , войди.
2. https://openrouter.ai/keys → **Create Key** → скопируй (`sk-or-v1-...`).
3. Бесплатные модели: https://openrouter.ai/models?max_price=0 (помечены `:free`).

## 3️⃣ Установка и настройка

```bash

pip install -r requirements.txt

# 2. конфиги из образцов

```


Открой **`config.yaml`** и впиши **4 токена ботов** (поле `telegram_token` у
каждого) и проверь `boss.username` / `boss.telegram_id`.

> 🔐 `config.yaml` и `.env` уже в `.gitignore` — реальные токены не утекут в git.

## 4️⃣ Запуск

```bash
python orchestrator.py
```

Если что-то не заполнено — сервис **не упадёт молча**, а покажет понятный список,
что именно нужно дописать.

После запуска зайди в свою Telegram-группу и просто **напиши идею**, например:
> «Хочу приложение, которое помогает находить попутчиков для бега»

Боты начнут обсуждать её по кругам. Команды в чате:
- `/help` — справка
- `/stop` — остановить текущее обсуждение
- `/reset` — очистить память диалога
- `/status` — что сейчас происходит

## 🔁 Как менять модели и характеры

Всё — в `config.yaml`, секция `bots`. У каждого бота:
- `model` — любой id с https://openrouter.ai/models?max_price=0 (бесплатные `:free`).
  Если модель пропала/лимитит — впиши другую `:free`.
- `persona` — текст роли/характера. Меняй как хочешь, можно добавить/убрать ботов
  (минимум 2, рекомендуется 3–4).
- `listener: true` — ровно у одного бота (он слушает группу).

Параметры обсуждения — секция `discussion`:
- `max_rounds` — кругов на тему (по умолч. 3). Защита от бесконечного спама.
- `delay_seconds` — пауза между репликами (анти-спам и анти-rate-limit).
- `max_message_chars` — ботам велено писать кратко.

Модель-«модератор» (`moderator.model`) после каждого круга решает:
продолжить / задать тебе вопрос / подвести финальный итог.

## 🛡 Надёжность и заметки

- **Бесплатные модели OpenRouter** часто упираются в rate-limit/таймауты. Сервис
  делает повторы с бэкоффом, а если модель так и не ответила — **бот просто
  пропускает ход**, а не роняет всё приложение.
- Список бесплатных моделей у OpenRouter **меняется** — держи актуальные id в
  `config.yaml`.
- Транскрипт хранится **в памяти** (по чату). Перезапустил сервис — история
  обнулилась. Для прототипа этого достаточно.
- **Приоритет Боссу** зашит в системный промпт каждого бота: твои указания и
  ответы важнее мнения любого бота, и боты подстраиваются под них сразу.

## ⌨️ Анимация «печатает…», стриминг и реакции

Боты ведут себя «живо»:
- :keyboard: пока бот думает — в чате виден индикатор *«печатает…»* (`sendChatAction`);
- :writing_hand: ответ *появляется постепенно*, как будто бот печатает (текст растёт
  через `editMessageText` с курсором `▍`) — это потоковый вывод модели;
- :eyes: как только боты *увидели* твоё сообщение — ставят на него *реакцию* `👀`
  (`setMessageReaction`).

Настройки — в `config.yaml → discussion`: `typing`, `stream`, `edit_interval_seconds`,
`react_on_seen`, `seen_emoji`.

> ⚠️ *Про `sendMessageDraft` (черновик-стриминг):* этот метод Bot API работает
> **только в личных чатах**. В группе Telegram он отдаёт `TEXTDRAFT_PEER_INVALID`.
> Проверено вживую — поэтому в группе используется связка *typing + постепенный
> edit*, которая даёт тот же визуальный эффект «печати».
>
> ⚠️ *Чтобы реакцию ставил КАЖДЫЙ бот* (а не только один): у **всех 4** ботов
> должен быть отключён Privacy Mode (`@BotFather → /setprivacy → Disable`) **или**
> они должны быть **админами** группы. Иначе бот не «видит» сообщение и поставить
> реакцию не может (в логах: `message to react not found`).

## 🛟 Если боты молчат (Troubleshooting)

Прототип отвечает **только пока запущен процесс** `python orchestrator.py`. Если
он не запущен или упал — боты молчат. Запускай его у себя или на VPS постоянно.

Частые ошибки от OpenRouter (видно в логах сервиса):

| Ошибка | Что значит | Как починить |
|--------|-----------|--------------|
| `404 No endpoints ... data policy` | Аккаунт запрещает free-провайдеров, которые могут обучаться на запросах. Блокирует gpt-oss, gemma, kimi и др. | Включи тумблер на https://openrouter.ai/settings/privacy (разрешить такие модели). |
| `429 temporarily rate-limited` | Жёсткий лимит бесплатного тарифа. | Закинь $10 на баланс OpenRouter (лимит free → ~1000/день), **или** впиши дешёвую платную модель (id без `:free`), **или** перейди на Groq. |
| `модель вернула пустой текст` | Конкретная free-модель капризничает. | Поменяй её id на другую `:free` в `config.yaml`. |

Код переживает эти сбои корректно: при ошибке модели бот **пропускает ход**, а не
роняет сервис. Поэтому при сплошных 429/404 в чате будет тишина (а не краш).

### ✅ Рекомендация для бесплатного варианта: Groq (быстро и без жёстких лимитов)

Код OpenAI-совместимый, поэтому Groq подключается **без переписывания** — только 3 правки.

1. Получи бесплатный ключ: https://console.groq.com/keys
2. В **`.env`** впиши его (раскомментируй строку):
   ```env
   GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
3. В **`config.yaml`** поставь Groq `base_url` и Groq-id моделей:
   ```yaml
   llm:
     base_url: "https://api.groq.com/openai/v1"
   moderator:
     model: "llama-3.1-8b-instant"
   bots:
     - name: "Стратег"   →  model: "openai/gpt-oss-120b"
     - name: "Креатор"   →  model: "llama-3.3-70b-versatile"
     - name: "Критик"    →  model: "qwen/qwen3-32b"
     - name: "Реалист"   →  model: "openai/gpt-oss-20b"
   ```

Актуальные id моделей Groq всегда здесь: https://console.groq.com/docs/models
(на момент написания доступны: `openai/gpt-oss-120b`, `openai/gpt-oss-20b`,
`llama-3.3-70b-versatile`, `llama-3.1-8b-instant`, `qwen/qwen3-32b`).

> Запасной ключ из `GROQ_API_KEY` подхватывается автоматически — отдельная секция
> в коде не нужна. Если в `.env` заданы и `OPENROUTER_API_KEY`, и `GROQ_API_KEY`,
> приоритет у OpenRouter (удали лишний, чтобы не путаться).

## ☁️ Хостинг без своего ПК

### Вариант А — GitHub Actions: сессия «по кнопке»

Workflow поднимает ботов на заданное число минут и сам выключает их.

> ⚠️ Файл workflow нужно создать вручную (GitHub не даёт ботам/приложениям
> добавлять файлы в `.github/workflows/` за тебя). Это 1 минута:
> на GitHub нажми **Add file → Create new file**, путь
> `.github/workflows/brainstorm.yml`, вставь YAML ниже и закоммить.

<details>
<summary>📄 Содержимое <code>.github/workflows/brainstorm.yml</code> (скопировать)</summary>

```yaml
name: Brainstorm session (по кнопке)

on:
  workflow_dispatch:
    inputs:
      minutes:
        description: "Сколько минут держать ботов онлайн (1–350; лимит GitHub ~6ч)"
        required: true
        default: "120"

jobs:
  run:
    runs-on: ubuntu-latest
    timeout-minutes: 355
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r requirements.txt
      - name: Prepare config (Groq, без секретов в файле)
        run: cp config.actions.yaml config.yaml
      - name: Run brainstorm orchestrator
        env:
          GROQ_API_KEY:      ${{ secrets.GROQ_API_KEY }}
          BOT_TOKEN_STRATEG: ${{ secrets.BOT_TOKEN_STRATEG }}
          BOT_TOKEN_KREATOR: ${{ secrets.BOT_TOKEN_KREATOR }}
          BOT_TOKEN_KRITIK:  ${{ secrets.BOT_TOKEN_KRITIK }}
          BOT_TOKEN_REALIST: ${{ secrets.BOT_TOKEN_REALIST }}
        run: |
          MIN="${{ github.event.inputs.minutes }}"
          echo "Запуск ботов на ${MIN} минут…"
          timeout "${MIN}m" python orchestrator.py || true
          echo "Сессия завершена."
```
</details>

> ⚠️ *Actions — НЕ постоянный 24/7-сервер.* Один запуск ограничен ~6 часами,
> а запуск по расписанию (cron) ненадёжен (задержки/пропуски). Поэтому Actions
> годится для **«побрейнштормили и закрыли»**, но не для «бот всегда онлайн».

> ⚠️ *Главное:* Actions работает **на серверах GitHub, а не на твоём ПК** — твоего
> локального `.env` там НЕТ. Токены и ключ нужно положить в **GitHub Actions Secrets**
> (это бесплатно и не светится в коде). Коммитить реальный `.env` в публичный репо
> НЕЛЬЗЯ — токены утекут в историю git.

*Шаги:*
1. В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.
   Создай 5 секретов (значения — твои реальные токены/ключ):

   | Имя секрета | Значение |
   |---|---|
   | `GROQ_API_KEY` | ключ Groq (`gsk_...`) |
   | `BOT_TOKEN_STRATEG` | токен бота Стратег |
   | `BOT_TOKEN_KREATOR` | токен бота Креатор |
   | `BOT_TOKEN_KRITIK` | токен бота Критик |
   | `BOT_TOKEN_REALIST` | токен бота Реалист |

2. Вкладка **Actions** → workflow **«Brainstorm session (по кнопке)»** → **Run workflow**
   → укажи минуты (напр. 120) → запустить.
3. Пока job идёт — пиши идею в группу, боты отвечают. По истечении минут (или ~6ч)
   сессия сама закрывается. Конфиг для Actions — `config.actions.yaml` (Groq, без секретов).

### Вариант Б — бесплатный хост «всегда онлайн» (если нужно 24/7)
