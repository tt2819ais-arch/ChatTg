# 🤖 Telegram Brainstorm Bots — команда ИИ, которая штурмует твою идею

Прототип: групповой чат в Telegram, где **3–4 ИИ-бота** с разными характерами
(Стратег, Креатор, Критик, Реалист) обсуждают идею, которую ты кидаешь в чат.
Они спорят по кругам, задают тебе уточняющие вопросы и **слушаются тебя в
первую очередь**. Когда идея доведена до ума — подводят итог.

---

## ⚠️ Главное про Telegram (почему всё устроено так)

Бот в Telegram **НЕ получает сообщения, которые отправил другой бот**. То есть
боты физически не могут «слышать» друг друга через Telegram и переписываться
сами.

Поэтому здесь **один центральный Python-сервис** держит все 4 токена сразу:
- ведёт **один общий транскрипт** диалога;
- по очереди спрашивает каждый ИИ, выдавая ему всю историю + его личную роль;
- постит каждую реплику в группу **под токеном нужного бота**.

Так в чате выглядит, будто 4 бота переписываются между собой, хотя на самом
деле их «мозгом» и «дирижёром» управляет один сервис. Слушать сообщения группы
достаточно **одному** боту (`listener: true`) — остальные только отправляют.

## 1️⃣ Создаём 4 ботов у @BotFather

В Telegram открой **@BotFather** и для **каждого** из 4 ботов:

1. `/newbot` → задай имя и username (напр. `My Strateg Bot`, `my_strateg_bot`).
2. BotFather пришлёт **токен** вида `123456:ABC-...` — сохрани его.
3. **Отключи приватный режим**, чтобы бот видел сообщения группы:
   `/setprivacy` → выбери этого бота → **Disable**.
4. Повтори для всех 4 ботов.

Затем:
- Создай **группу** в Telegram, добавь в неё **все 4 бота** и **себя**.
- Сделай ботов **админами** группы (по желанию, но так надёжнее — особенно если
  приватный режим где-то остался включён).

> 💡 Свой числовой Telegram **id** узнаешь у @userinfobot — он уже прописан в
> конфиге (`boss.telegram_id`), но проверь, что он правильный.

## 2️⃣ Берём ключ OpenRouter

1. Зайди на https://openrouter.ai , войди.
2. https://openrouter.ai/keys → **Create Key** → скопируй (`sk-or-v1-...`).
3. Бесплатные модели: https://openrouter.ai/models?max_price=0 (помечены `:free`).

## 3️⃣ Установка и настройка

```bash
# 1. зависимости
pip install -r requirements.txt

# 2. конфиги из образцов
cp config.example.yaml config.yaml
cp .env.example .env
```

Открой **`.env`** и впиши ключ OpenRouter:
```
OPENROUTER_API_KEY=sk-or-v1-твой_ключ
```

Открой **`config.yaml`** и впиши **4 токена ботов** (поле `telegram_token` у
каждого) и проверь `boss.username` / `boss.telegram_id`.

> 🔐 `config.yaml` и `.env` уже в `.gitignore` — реальные токены не утекут в git.

## 4️⃣ Запуск

```bash
python orchestrator.py
```

Если что-то не заполнено — сервис **не упадёт молча**, а покажет понятный список,
что именно нужно дописать.

После запуска зайди в свою Telegram-группу и просто **напиши идею**, например:
> «Хочу приложение, которое помогает находить попутчиков для бега»

Боты начнут обсуждать её по кругам. Команды в чате:
- `/help` — справка
- `/stop` — остановить текущее обсуждение
- `/reset` — очистить память диалога
- `/status` — что сейчас происходит

## 🔁 Как менять модели и характеры

Всё — в `config.yaml`, секция `bots`. У каждого бота:
- `model` — любой id с https://openrouter.ai/models?max_price=0 (бесплатные `:free`).
  Если модель пропала/лимитит — впиши другую `:free`.
- `persona` — текст роли/характера. Меняй как хочешь, можно добавить/убрать ботов
  (минимум 2, рекомендуется 3–4).
- `listener: true` — ровно у одного бота (он слушает группу).

Параметры обсуждения — секция `discussion`:
- `max_rounds` — кругов на тему (по умолч. 3). Защита от бесконечного спама.
- `delay_seconds` — пауза между репликами (анти-спам и анти-rate-limit).
- `max_message_chars` — ботам велено писать кратко.

Модель-«модератор» (`moderator.model`) после каждого круга решает:
продолжить / задать тебе вопрос / подвести финальный итог.

## 🛡 Надёжность и заметки

- **Бесплатные модели OpenRouter** часто упираются в rate-limit/таймауты. Сервис
  делает повторы с бэкоффом, а если модель так и не ответила — **бот просто
  пропускает ход**, а не роняет всё приложение.
- Список бесплатных моделей у OpenRouter **меняется** — держи актуальные id в
  `config.yaml`.
- Транскрипт хранится **в памяти** (по чату). Перезапустил сервис — история
  обнулилась. Для прототипа этого достаточно.
- **Приоритет Боссу** зашит в системный промпт каждого бота: твои указания и
  ответы важнее мнения любого бота, и боты подстраиваются под них сразу.

## 🛟 Если боты молчат (Troubleshooting)

Прототип отвечает **только пока запущен процесс** `python orchestrator.py`. Если
он не запущен или упал — боты молчат. Запускай его у себя или на VPS постоянно.

Частые ошибки от OpenRouter (видно в логах сервиса):

| Ошибка | Что значит | Как починить |
|--------|-----------|--------------|
| `404 No endpoints ... data policy` | Аккаунт запрещает free-провайдеров, которые могут обучаться на запросах. Блокирует gpt-oss, gemma, kimi и др. | Включи тумблер на https://openrouter.ai/settings/privacy (разрешить такие модели). |
| `429 temporarily rate-limited` | Жёсткий лимит бесплатного тарифа. | Закинь $10 на баланс OpenRouter (лимит free → ~1000/день), **или** впиши дешёвую платную модель (id без `:free`), **или** перейди на Groq. |
| `модель вернула пустой текст` | Конкретная free-модель капризничает. | Поменяй её id на другую `:free` в `config.yaml`. |

Код переживает эти сбои корректно: при ошибке модели бот **пропускает ход**, а не
роняет сервис. Поэтому при сплошных 429/404 в чате будет тишина (а не краш).

### 🔁 Альтернатива: Groq (бесплатно, быстро, надёжнее для чат-ботов)

Код OpenAI-совместимый, так что Groq подключается без переписывания — поменяй
в `config.yaml` секцию `openrouter` и `model` у ботов:

```yaml
openrouter:
  api_key: ""                               # ключ Groq в .env: OPENROUTER_API_KEY=gsk_...
  base_url: "https://api.groq.com/openai/v1"
# и у ботов, например:
#   model: "llama-3.3-70b-versatile"
```
